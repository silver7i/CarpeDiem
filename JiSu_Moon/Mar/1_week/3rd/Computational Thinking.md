### 프로그래밍과 논리/수학
#### 논리(Hard Logic)
- 생각이나 주장의 타당성과 옳고 그름을 판별하는 규칙이나 원리
- 경험적, 직관적, 확률적 판단을 의미하는 Soft Logic에 반대되는 의미로 Hard Logic 이라고 표현

#### 명제
- 참이나 거짓을 알 수 있는 식이나 문장
```
- 명제 : 참과 거짓으로 판별할 수 있는 문장
  - 2는 짝수이다. -> 참인 명제
  - 3은 짝수이다. -> 거짓인 면제
  - 서울은 대한민국의 수도이다. -> 참인 명제
  - 강의는 재밌다. -> 명제가 아니다.
  - 강사님은 멋있다. -> 명제가 아니다.

- 명제는 보통 이런식으로 많이 사용된다.
  - p -> q (조건명제)
    - p가 참일 때, q도 참이다.

- 예시
- 4가 짝수일 때, 4^2 도 짝수이다.
```

### 논리와 증명 (명제를 증명하는 방법)
1. 대우 증명
2. 직접 증명
3. 간접 증명
```
문제 12: n2n^2n2 이 3의 배수이면 nnn은 3의 배수임을 증명하라.

Proof
자연수 n을 3으로 나누면 다음 세 가지 경우가 있다.

(1) n이 3의 배수인 경우
n = 3k (k ≥ 1)
n^2 = (3k)^2
    = 9k^2
    = 3(3k^2)
따라서 n^2은 3의 배수이다.

(2) n = 3k + 1 인 경우
n = 3k + 1
n^2 = (3k + 1)^2
    = 9k^2 + 6k + 1
    = 3(3k^2 + 2k) + 1
따라서 n^2은 3의 배수가 아니다.

(3) n = 3k + 2 인 경우
n = 3k + 2
n^2 = (3k + 2)^2
    = 9k^2 + 12k + 4
    = 3(3k^2 + 4k + 1) + 1
따라서 n^2은 3의 배수가 아니다.

결론
n이 3k+1 또는 3k+2이면 n^2은 3의 배수가 아니다.
따라서 n^2이 3의 배수이면 반드시 n = 3k 이다.
즉, n은 3의 배수이다.
```

### 기초 수식
시간 복잡도 (빅오표기법)
```
# 문제 4: T(n)=T(n/2)+1,T(1)=1
T(n) = T(n/2) + 1
= T(n/4) + 1 + 1
= T(n/8) + 1 + 1 + 1
= T(n/2^k) + k
= T(1) + log₂ n
= 1 + log₂ n

O(T(n)) = O(log n)
```
