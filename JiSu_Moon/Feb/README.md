### 월간 회고록
1. 정렬 (데이터 줄 세우기)
- 버블정렬 : 옆이랑 계속 비교하기
- 순차정렬 : 기준 잡고 뒤랑 다 비교하기 (즉시 자리 바꿈)
- 선택정렬 : 제일 작은 애 찍어서 한번에 바꾸기 (최솟값 찾아내면 바꿈)
- 계수정렬 : 숫자의 개수를 세어서 정렬
- 셀렉션 알고리즘 : 다 정렬하지 않고 'k번째로 작은 값'만 딱 골라내기

2. 검색 (원하는 데이터 찾기)
- 순차검색 : 정렬 불필요, 속도 느림 (데이터 수만큼 확인)
- 이진검색 : 정렬 필요, 속도 빠름 (로그 스케일)
- 파라메트릭 서치 : 정렬 불필요(상태는 반드시 정렬), 속도 매우 빠름, 이진 검색 원리로 "조건을 만족하는 최적의 값" 찾기

3. 경우의 수 만들기 (완전 탐색의 재료)
- 순열: 순서 있게 나열하기 (A-B ≠ B-A)
- 조합: 순서 상관없이 뽑기 (A-B = B-A)
- 중복 순열/중복 조합: 이미 뽑은 걸 또 뽑을 수 있는 경우
- 부분집합: 각 원소를 넣을지 말지 결정하기 ($2^n$가지)

4. 탐색 전략 (문제 해결의 기술)
- Function Call(재귀) : 자기 자신을 호출해 문제를 쪼개기
- 분할정복 : 큰 문제를 작은 조각으로 나눠서 풀고 합치기 (분할-정복-결합)
- DFS(깊이 우선) : Stack(스택) 또는 재귀 사용, 한 길만 끝까지 파보기 (미로 찾기)
- BFS(너비 우선) : Queue(큐) 사용, 주변부터 넓게 퍼지며 찾기 (최단 거리)

5. 최적화 전략 (효율성 극대화)
- 완전 검색: 무식하지만 확실하게 모든 경우의 수를 다 해보기.
- 백트래킹 (가지치기): 가보다가 답이 없으면 중간에 포기하고 되돌아오기.
- 그리디 (탐욕): 당장 눈앞의 최선의 선택만 하기 (동전)
- DP (동적 계획법) / Memoization: DAT와 비슷한 원리(저장 후 재사용)
